# 🚀 Getting Started

## 1. Fork the Repository

Click the **Fork** button at the top-right corner of the GitHub repository.

---

## 2. Clone Your Fork

```bash
git clone https://github.com/your-username/AI-SQL-Agent.git
```

Navigate into the project:

```bash
cd AI-SQL-Agent
```

---

## 3. Create a Virtual Environment

```bash
python -m venv .venv
```

Activate the environment.

### Windows

```bash
.venv\Scripts\activate
```

### Linux/macOS

```bash
source .venv/bin/activate
```

---

## 4. Install Dependencies

```bash
pip install -r requirements.txt
```

---

## 5. Run the Project

```bash
streamlit run app.py
```

Verify the application starts successfully before making changes.

---

# 🌿 Branch Naming

Create a new branch for every contribution.

Examples:

```text
feature/dashboard-improvements

feature/query-templates

feature/pdf-export

bugfix/mongo-validator

bugfix/chart-rendering

docs/readme-update

refactor/orchestrator
```

Do **not** commit directly to the `main` branch.

---

# 💻 Coding Guidelines

Please follow these guidelines while contributing:

- Write clean, readable code.
- Keep functions small and focused.
- Use meaningful variable and function names.
- Add comments only when necessary.
- Follow existing project structure.
- Avoid duplicate code.
- Test your changes before submitting.

---

# 🧪 Testing

Before creating a Pull Request, ensure:

- The application starts successfully.
- SQL queries execute correctly.
- MongoDB queries execute correctly.
- Dashboard loads properly.
- Export features work correctly.
- Charts display correctly.
- No existing functionality is broken.

# 📝 Commit Messages

Use descriptive commit messages.

Examples:

```text
Add MongoDB Repair Agent

Improve Dashboard Analytics

Fix SQL Validator bug

Add PDF Export feature

Update README documentation

Improve Query Templates
```

Avoid generic commit messages such as:

```text

changes

fixed

work

commit
```

# 🔄 Pull Request Process

1. Fork the repository.
2. Create a feature branch.
3. Commit your changes.
4. Push the branch.
5. Open a Pull Request.
6. Describe your changes clearly.
7. Wait for review and feedback.

# 🐛 Reporting Issues

If you discover a bug, please create a GitHub Issue with:

- Bug description
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Environment details

Example:

```text
Python Version: 3.11

Operating System: Windows 11

Database: MySQL

Provider: Ollama

Error:
...
```
